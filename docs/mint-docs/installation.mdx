# Installation Guide

Complete setup instructions for Average At Best: Pokemon Battle League.

## System Requirements

- **Node.js**: 22.0.0 or higher
- **pnpm**: 8.0.0 or higher
- **PostgreSQL**: 14+ (via Supabase)
- **Git**: Latest version

## Step-by-Step Installation

### 1. Clone Repository

```bash
git clone https://github.com/moodmnky/POKE-MNKY-v2.git
cd POKE-MNKY-v2
```

### 2. Install Dependencies

```bash
pnpm install
```

### 3. Environment Setup

Copy .env.example to .env.local and fill in your values:

```bash
cp .env.example .env.local
```

Required environment variables:

- `NEXT_PUBLIC_SUPABASE_URL` - Your Supabase project URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` - Supabase anonymous key
- `SUPABASE_SERVICE_ROLE_KEY` - Supabase service role key
- `OPENAI_API_KEY` - OpenAI API key for AI features

Optional variables:

- `DISCORD_BOT_TOKEN` - Discord bot token
- `DISCORD_CLIENT_ID` - Discord OAuth client ID
- `DISCORD_CLIENT_SECRET` - Discord OAuth client secret
- `NOTION_SYNC_SECRET` - Secret for Notion sync endpoints

### 4. Database Setup

#### Option A: Supabase Cloud

1. Create a new Supabase project
2. Run migrations from `supabase/migrations/` in SQL Editor
3. Configure Row Level Security policies

#### Option B: Local Supabase

```bash
# Install Supabase CLI
npm install -g supabase

# Start local Supabase
supabase start

# Link to project (if using cloud)
supabase link --project-ref your-project-ref

# Push migrations
supabase db push
```

### 5. Run Development Server

```bash
pnpm dev
```

Visit `http://localhost:3000` to see the app.

## Verification

### Check Database Connection

```bash
pnpm db:sync:pre
```

### Verify Migrations

Check Supabase dashboard → Database → Migrations to ensure all migrations ran successfully.

### Test API Endpoints

```bash
curl http://localhost:3000/api/health
```

## Troubleshooting

### Common Issues

**Port Already in Use**
- Change port: `pnpm dev -- -p 3001`

**Database Connection Errors**
- Verify Supabase URL and keys
- Check network connectivity
- Ensure migrations ran successfully

**Missing Dependencies**
- Run `pnpm install` again
- Clear `node_modules` and reinstall
